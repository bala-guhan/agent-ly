TECH STACK:

FRONTEND:
- Framework: React 18, Next.js 14 (for SSR needs)
- Language: TypeScript (strict mode)
- State Management: Redux Toolkit, React Query (server state)
- Styling: Tailwind CSS, shadcn/ui component library
- Forms: React Hook Form, Zod validation
- Testing: Jest, React Testing Library, Cypress (E2E)
- Build: Vite, Webpack (legacy parts)

BACKEND:
- Primary Language: Node.js (v20 LTS), Express.js
- Secondary Language: Python 3.11 (FastAPI for ML services)
- API: RESTful APIs, GraphQL (for complex queries)
- Authentication: OAuth 2.0, JWT, SAML 2.0
- Background Jobs: Bull (Redis-based queue)
- Caching: Redis (primary), Memcached (legacy)

DATABASES:
- Primary DB: PostgreSQL 15 (Amazon RDS Multi-AZ)
- Document Store: MongoDB (for unstructured clinical notes)
- Time-Series: TimescaleDB (for vitals, monitoring data)
- Search: Elasticsearch 8 (for full-text search across patient records)
- Data Warehouse: Snowflake (for analytics and reporting)

INFRASTRUCTURE & CLOUD:
- Cloud Provider: AWS (primary), Google Cloud (DR site)

DEVELOPMENT PRACTICES:

AGILE METHODOLOGY:
- Sprint Length: 2 weeks
- Sprint Planning: Monday morning (2 hours)
- Daily Standup: 9:30 AM IST (15 minutes, async on Slack for distributed team)
- Sprint Review: Friday afternoon (1 hour)
- Sprint Retrospective: Friday afternoon (1 hour)
- Refinement: Mid-sprint Wednesday (1 hour)

CODE REVIEW PROCESS:
- All code requires 2 approvals before merge (1 from squad, 1 from outside squad)
- PR size limit: <500 lines of code (enforced by CI bot)
- Automated checks: Linting, unit tests, security scans must pass
- Review SLA: 24 hours for first review, 48 hours for approval
- Architecture Decision Records (ADRs) required for significant changes

TESTING REQUIREMENTS:
- Unit Test Coverage: Minimum 80% for backend, 70% for frontend
- Integration Tests: Required for all API endpoints
- E2E Tests: Critical user flows (patient registration, prescription, billing)
- Performance Tests: Run weekly on staging, before major releases
- Security Tests: OWASP ZAP scans, dependency vulnerability scans (Snyk)

DEPLOYMENT PROCESS:
- Staging Deployment: Automatic on merge to develop branch
- Production Deployment: Tuesdays and Thursdays only (avoid Mondays/Fridays)
- Deployment Window: 10 PM - 2 AM IST (low traffic period)
- Canary Releases: 10% traffic → 50% → 100% over 3 hours
- Rollback Plan: One-click rollback available for 48 hours post-deployment
- Feature Flags: LaunchDarkly for gradual feature rollout

ON-CALL ROTATION:
- Primary On-Call: 1-week rotation across all senior engineers
- Secondary On-Call: Backup for escalations
- Incidents: Page for P0 (critical outage), Slack alert for P1 (degraded service)
- Incident Response: War room on Google Meet, postmortem within 48 hours
- On-Call Compensation: ₹10,000/week on-call allowance + comp time for incidents

DOCUMENTATION:
- Technical Docs: Maintained in Confluence
- API Documentation: Auto-generated from OpenAPI specs, hosted on docs.2care.health
- Runbooks: For common operational tasks, stored in GitHub
- Architecture Diagrams: C4 model, updated quarterly
- Onboarding Docs: 2-week onboarding plan for new engineers

SECURITY PRACTICES:
- Code Scanning: SonarQube for code quality, Checkmarx for SAST
- Dependency Scanning: Dependabot, Snyk for vulnerability alerts
- Secrets: No secrets in code (enforced by pre-commit hooks)
- Access Control: Least privilege principle, annual access reviews
- Security Training: Mandatory OWASP training for all engineers annually
- Penetration Testing: External pentests twice yearly

PERFORMANCE BENCHMARKS:
- API Response Time: p95 < 500ms, p99 < 1 second
- Web App Load Time: First Contentful Paint < 1.5s
- Database Query Time: 95% of queries < 100ms
- Deployment Frequency: 15-20 production deployments per month
- Change Failure Rate: Target < 5%
- Mean Time to Recovery (MTTR): Target < 1 hour for P0 incidents

INNOVATION TIME:
- 10% Time: Engineers can spend up to 4 hours/week on learning or side projects
- Hackathons: Quarterly 2-day internal hackathons
- Tech Talks: Weekly Friday tech talks (30 minutes, optional)
- Conference Attendance: Budget for 2 conferences/year per engineer
- Open Source: Encouraged to contribute (with approval for healthcare-related projects)

HIRING & GROWTH:
- Hiring Plan 2025: Add 45 engineers (24% growth)
- Interview Process: Coding round (90 min) → System design (90 min) → Behavioral (60 min) → Hiring manager (45 min)
- Offer Timeline: Decision within 48 hours of final interview
- Career Ladder: IC track (Engineer → Senior → Staff → Principal) and Management track
- Promotion Cycle: Twice yearly (March, September)
- Internal Mobility: Encouraged after 18 months in current role
